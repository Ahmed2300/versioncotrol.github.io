رحلة عملية: بناء مشروع متكامل بـ Git

سنطبق كل الأوامر الأساسية خطوة بخطوة من خلال بناء مشروع حقيقي.

المرحلة الأولى: الإعداد والتهيئة

1. فحص تثبيت Git وطلب المساعدة

الأمر: git
الوظيفة: عند كتابته بمفرده، يعرض قائمة بالأوامر الأكثر شيوعًا. هذه أسرع طريقة للتأكد من أن Git مثبت ويعمل على جهازك.
git
الأمر: git --help
الوظيفة: يفتح دليل المساعدة الشامل لـ Git في متصفحك، وهو مرجعك الأول عند الحاجة لمعلومات تفصيلية.
git --help

2. إعداد معلومات المستخدم (Config)

هذه الخطوة ضرورية ويجب فعلها مرة واحدة فقط على جهازك. تحدد هويتك في كل commit تقوم به.

الأمر: git config --global user.name "Your Name"
الوظيفة: يقوم بتعيين اسمك على المستوى العام (Global)، ليتم استخدامه في كل مستودعات Git على جهازك.
git config --global user.name "Ahmed Azam"
الأمر: git config --global user.email "your.email@example.com"
الوظيفة: يقوم بتعيين بريدك الإلكتروني على المستوى العام.
git config --global user.email "ahmed@example.com"
الأمر: git config --list
الوظيفة: يعرض جميع إعدادات Git (System, Global, Local) للتحقق من أن الإعدادات تم حفظها بنجاح.
git config --list

3. إنشاء المشروع وتهيئته

سنستخدم أوامر shell لإنشاء مجلد المشروع، ثم نهيئ Git بداخله.

cd Desktop
mkdir my-website
cd my-website
الأمر: git init
الوظيفة: يقوم بإنشاء مستودع Git فارغ في المجلد الحالي. ستلاحظ ظهور مجلد مخفي جديد باسم .git.
git init

المرحلة الثانية: دورة العمل الأساسية

1. إنشاء الملفات والتحقق من الحالة

الأمر: touch <file_name>
الوظيفة: أمر shell لإنشاء ملف جديد فارغ. سننشئ ملفين.
touch index.html styles.css
الأمر: git status
الوظيفة: يعرض حالة المستودع. سيظهر الملفان الجديدان باللون الأحمر تحت قسم "Untracked files".
git status

2. إضافة التغييرات إلى منطقة التجهيز (Staging)

يمكنك إضافة الملفات إلى منطقة التجهيز بعدة طرق. يمكنك إضافتها كل على حدة، أو تحديد عدة ملفات، أو إضافة كل التغييرات دفعة واحدة.

الطريقة الأولى: إضافة عدة ملفات محددة
الأمر: git add index.html styles.css
الوظيفة: يضيف الملفين المحددين إلى منطقة التجهيز.
git add index.html styles.css

بعد تنفيذ الأمر، لنتحقق من الحالة مرة أخرى.

git status
النتيجة المتوقعة: ستلاحظ أن كلا الملفين أصبحا باللون الأخضر، مما يعني أنهما جاهزان للـ commit. 
On branch main

Changes to be committed:
  (use "git restore --staged ..." to unstage)
        new file:   index.html
        new file:   styles.css
الطريقة الثانية: إضافة كل التغييرات دفعة واحدة

إذا كان لديك العديد من الملفات الجديدة أو المعدلة، فإن استخدام git add . هو الطريقة الأسرع لإضافتها جميعًا إلى منطقة التجهيز.

الأمر: git add .
الوظيفة: يضيف كل التغييرات الجديدة والمعدلة في المجلد الحالي وتفرعاته إلى منطقة التجهيز.
git add .

3. حفظ التغييرات في المستودع (Commit)

الأمر: git commit -m "Commit message"
الوظيفة: يأخذ كل ما في منطقة التجهيز ويحفظه بشكل دائم في تاريخ المستودع مع رسالة وصفية.
git commit -m "feat: Create initial project files"

4. عرض التاريخ والفروقات

الأمر: git log
الوظيفة: يعرض تاريخ الـ commits بالتفصيل (الهاش، المؤلف، التاريخ، الرسالة).
git log
الأمر: git log --oneline
الوظيفة: يعرض التاريخ بشكل مختصر، كل commit في سطر واحد.
git log --oneline

الآن، لنقم بتعديل ملف `index.html` ونرى الفرق.

الأمر: git diff
الوظيفة: يعرض الفروقات بين ملفات مجلد العمل (Working Directory) وآخر commit.
git diff

المرحلة الثالثة: التفرع والدمج

التفرع (Branching) هو أحد أقوى المفاهيم في Git. يسمح لك بإنشاء خط تطوير مستقل ومنعزل عن الخط الرئيسي (master). يمكنك العمل على ميزة جديدة أو إصلاح خطأ دون التأثير على الكود المستقر. الدمج (Merging) هو عملية إعادة توحيد الفرع المنفصل مع الخط الرئيسي بعد اكتمال العمل.

1. إنشاء الفروع والتنقل بينها

الأمر: git branch
الوظيفة: يعرض قائمة بكل الفروع المحلية. الفرع الحالي سيظهر بجانبه علامة (*).
git branch
الأمر: git branch <new_branch_name>
الوظيفة: يقوم بإنشاء فرع جديد ولكنه لا ينتقل إليه.
git branch feature/add-header
الأمر: git checkout <existing_branch_name>
الوظيفة: ينتقل إلى فرع موجود بالفعل.
git checkout feature/add-header

هناك طريقة مختصرة لإنشاء فرع والانتقال إليه في خطوة واحدة:

الأمر: git checkout -b <new_branch_name>
الوظيفة: يقوم بإنشاء فرع جديد وينتقل إليه فورًا.
git checkout -b fix/css-bug

2. الدمج (Merge)

بعد الانتهاء من العمل على فرع `feature/add-header`، نريد دمج تغييراته في الفرع الرئيسي `main`.

git checkout main
الأمر: git merge <branch_to_merge>
الوظيفة: يقوم بدمج التغييرات من الفرع المحدد في الفرع الحالي الذي تقف عليه.
git merge feature/add-header

لنرَ كيف يبدو التاريخ الآن مع عملية الدمج:

الأمر: git log --oneline --graph
الوظيفة: يعرض التاريخ بشكل مختصر مع رسم بياني يوضح مسارات الفروع وعمليات الدمج.
